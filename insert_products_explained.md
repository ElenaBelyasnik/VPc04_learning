# Вставка товаров в Products: объяснение и примеры

Этот документ объясняет, как вставлять данные в таблицу `Products` с примерами ручной и массовой вставки.

---

## Структура таблицы Products

```sql
CREATE TABLE Products (
    ID INTEGER PRIMARY KEY AUTOINCREMENT,
    Name TEXT NOT NULL,
    Description TEXT,
    Category TEXT,
    Price REAL,
    SKU TEXT UNIQUE
);
```

### Пояснение колонок:

| Колонка | Тип | Описание |
|---------|-----|----------|
| `ID` | INTEGER | Уникальный идентификатор (автоинкремент) |
| `Name` | TEXT | Название товара (обязательно) |
| `Description` | TEXT | Описание товара (может быть NULL) |
| `Category` | TEXT | Категория товара |
| `Price` | REAL | Цена товара |
| `SKU` | TEXT | Уникальный артикул (Stock Keeping Unit) |

---

## 1. Ручная вставка (INSERT INTO ... VALUES)

### Синтаксис:

```sql
INSERT INTO Products (Name, Description, Category, Price, SKU)
VALUES ('Название', 'Описание', 'Категория', 999.99, 'ART-12345-A');
```

### Пример 1: Вставка товара "Яблоки"

```sql
INSERT INTO Products (Name, Description, Category, Price, SKU)
VALUES ('Яблоки', 'Свежие яблоки, 1 кг', 'Продукты', 500.00, 'ART-99999-A');
```

### Пример 2: Вставка товара "Бананы"

```sql
INSERT INTO Products (Name, Description, Category, Price, SKU)
VALUES ('Бананы', 'Спелые бананы, 1 кг', 'Продукты', 120.50, 'ART-99998-B');
```

### Пример 3: Вставка товара "Молоко"

```sql
INSERT INTO Products (Name, Description, Category, Price, SKU)
VALUES ('Молоко', 'Свежее молоко 3.2%, 1 л', 'Продукты', 89.90, 'ART-99997-C');
```

### Пояснение синтаксиса:

| Часть запроса | Описание |
|---------------|----------|
| `INSERT INTO Products` | Указывает таблицу для вставки |
| `(Name, Description, ...)` | Список колонок для вставки |
| `VALUES` | Ключевое слово перед значениями |
| `('...', '...', ...)` | Значения в том же порядке, что и колонки |
| `;` | Окончание SQL-запроса |

## 2. Массовая вставка (Multiple VALUES)

### Синтаксис:

```sql
INSERT INTO Products (Name, Description, Category, Price, SKU)
VALUES 
    ('Товар 1', 'Описание 1', 'Категория', 100.00, 'SKU-001'),
    ('Товар 2', 'Описание 2', 'Категория', 200.00, 'SKU-002'),
    ('Товар 3', 'Описание 3', 'Категория', 300.00, 'SKU-003');
```

### Полный пример:

```sql
INSERT INTO Products (Name, Description, Category, Price, SKU)
VALUES 
    ('Молоко', 'Свежее молоко 3.2%, 1 л', 'Продукты', 89.90, 'ART-99997-C'),
    ('Хлеб', 'Белый хлеб, 400 г', 'Продукты', 45.00, 'ART-99996-D'),
    ('Сыр', 'Твёрдый сыр, 200 г', 'Продукты', 250.00, 'ART-99995-A'),
    ('Куриное филе', 'Охлаждённое, 1 кг', 'Продукты', 350.00, 'ART-99994-B'),
    ('Рис', 'Длиннозёрный, 1 кг', 'Продукты', 95.00, 'ART-99993-C');
```

### Пояснение:

| Часть запроса | Описание |
|---------------|----------|
| `VALUES` | Начало списка кортежей |
| `(...), (...), (...)` | Каждый кортеж — одна строка |
| `,` (запятая) | Разделитель между кортежами |
| **Последний без запятой** | После последнего значения запятая не ставится |

### Преимущества массовой вставки:

1. **Быстрее** — один запрос вместо 5 отдельных
2. **Эффективнее** — меньше обращений к БД
3. **Чище код** — проще читать и поддерживать

## 3. Вставка через Python (executemany)

### Пример массовой вставки из Python:

```python
import sqlite3

conn = sqlite3.connect(r'C:\projects\Modul-3\VPc04_learning\learning.db')
cursor = conn.cursor()

# Список товаров для вставки
products = [
    ('Молоко', 'Свежее молоко 3.2%, 1 л', 'Продукты', 89.90, 'ART-99997-C'),
    ('Хлеб', 'Белый хлеб, 400 г', 'Продукты', 45.00, 'ART-99996-D'),
    ('Сыр', 'Твёрдый сыр, 200 г', 'Продукты', 250.00, 'ART-99995-A'),
    ('Куриное филе', 'Охлаждённое, 1 кг', 'Продукты', 350.00, 'ART-99994-B'),
    ('Рис', 'Длиннозёрный, 1 кг', 'Продукты', 95.00, 'ART-99993-C'),
]

# Массовая вставка через executemany
cursor.executemany("""
    INSERT INTO Products (Name, Description, Category, Price, SKU)
    VALUES (?, ?, ?, ?, ?)
""", products)

conn.commit()
print(f"✓ Вставлено {len(products)} товаров")
conn.close()
```

### Пояснение:

| Часть кода | Описание |
|------------|----------|
| `executemany()` | Метод для массовой вставки |
| `?` | Плейсхолдеры для параметров (защита от SQL-инъекций) |
| `products` | Список кортежей с данными |

## 5. Проверка вставленных данных

### Посчитать количество товаров:

```sql
SELECT COUNT(*) AS total_products FROM Products;
```

### Показать все товары:

```sql
SELECT * FROM Products;
```

### Показать товары из категории "Продукты":

```sql
SELECT Name, Price, SKU
FROM Products
WHERE Category = 'Продукты';
```

## 6. Часто встречающиеся ошибки

### ❌ Ошибка: дубликат SKU

```sql
-- Если SKU уже существует
INSERT INTO Products (Name, Price, SKU)
VALUES ('Товар', 100.00, 'ART-12345-A');

-- Ошибка: UNIQUE constraint failed: Products.SKU
```

**Решение:** Используйте уникальные SKU.

### ❌ Ошибка: пропущено обязательное поле

```sql
-- Name — обязательное поле (NOT NULL)
INSERT INTO Products (Price, SKU)
VALUES (100.00, 'ART-12345-A');

-- Ошибка: column Name cannot be null
```

**Решение:** Указывайте все NOT NULL колонки.

### ❌ Ошибка: неправильный порядок значений

```sql
-- Порядок не совпадает с колонками
INSERT INTO Products (Name, Price, SKU)
VALUES (100.00, 'Товар', 'ART-12345-A');

-- Ошибка: некорректный тип данных
```

**Решение:** Порядок значений должен совпадать с порядком колонок.

## 7. Рекомендации

| Рекомендация | Описание |
|--------------|----------|
| **Используйте массовую вставку** | Быстрее, чем много отдельных INSERT |
| **Проверяйте уникальность SKU** | Используйте `UNIQUE` constraint |
| **Используйте плейсхолдеры `?`** | Защита от SQL-инъекций в Python |
| **Округляйте цены** | `ROUND(Price, 2)` для корректного формата |
| **Проверяйте данные после вставки** | `SELECT COUNT(*)` для подтверждения |
