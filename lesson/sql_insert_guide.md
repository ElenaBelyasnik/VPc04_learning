## Структура таблицы Products

```sql
CREATE TABLE Products (
    ID INTEGER PRIMARY KEY AUTOINCREMENT,
    Name TEXT NOT NULL,
    Description TEXT,
    Category TEXT,
    Price REAL,
    SKU TEXT UNIQUE
);
```

### Пояснение каждого оператора и типа данных:

| Оператор / Тип | Описание |
|----------------|----------|
| `CREATE TABLE` | Создаёт новую таблицу в базе данных |
| `Products` | Имя таблицы (на английском, латиницей) |
| `ID` | Имя колонки для уникального идентификатора |
| `INTEGER` | Тип данных — целое число |
| `PRIMARY KEY` | Первичный ключ — уникально идентифицирует каждую строку |
| `AUTOINCREMENT` | Автоинкремент — значение увеличивается автоматически на 1 |
| `Name` | Имя колонки для названия товара |
| `TEXT` | Тип данных — строка произвольной длины |
| `NOT NULL` | Ограничение — значение обязательно должно быть указано |
| `Description` | Имя колонки для описания товара (может быть NULL) |
| `Category` | Имя колонки для категории товара (может быть NULL) |
| `Price` | Имя колонки для цены |
| `REAL` | Тип данных — число с плавающей точкой (дробное число) |
| `SKU` | Stock Keeping Unit — уникальный артикул/код товара |
| `UNIQUE` | Ограничение — все значения в колонке должны быть уникальными |

## Базовый оператор INSERT

### Синтаксис вставки одной строки:

```sql
INSERT INTO Products (Name, Description, Category, Price, SKU)
VALUES ('Беспроводные наушники', 'Высокое качество, гарантия 2 года', 'Электроника', 2999.99, 'ART-1234-A');
```

### Пояснение синтаксиса:

| Часть запроса | Описание |
|---------------|----------|
| `INSERT INTO` | Ключевые слова, указывающие на операцию вставки |
| `Products` | Имя таблицы, в которую вставляем данные |
| `(Name, Description, Category, Price, SKU)` | Список колонок для вставки (в этом порядке) |
| `VALUES` | Ключевое слово, отделяющее список колонок от значений |
| `('...', '...', '...', 99.99, '...')` | Значения для вставки в том же порядке, что и колонки |
| `;` | Окончание SQL-запроса |

### Важные правила:

1. **Порядок важен:** Значения должны идти в том же порядке, что и колонки.
2. **Типы данных:** Строки в кавычках (`'текст'`), числа без кавычек.
3. **NULL:** Если колонка допускает NULL, можно опустить значение или явно указать `NULL`.
4. **AUTOINCREMENT:** Колонку `ID` не нужно указывать — она заполняется автоматически.

## Массовая вставка через VALUES

### Синтаксис вставки нескольких строк за раз:

```sql
INSERT INTO Products (Name, Description, Category, Price, SKU)
VALUES 
    ('Беспроводные наушники', 'Высокое качество, гарантия 2 года', 'Электроника', 2999.99, 'ART-1234-A'),
    ('Смартфон XYZ', 'Мощный смартфон с камерой 108 МП', 'Электроника', 45999.00, 'ART-5678-B'),
    ('Книга по SQL', 'Полное руководство для начинающих', 'Книги', 899.50, 'ART-9012-C'),
    ('Офисное кресло', 'Эргономичное кресло с поддержкой спины', 'Мебель', 12499.00, 'ART-3456-D'),
    ('Набор отвёрток', 'Профессиональный набор на 50 предметов', 'Инструменты', 5699.99, 'ART-7890-A');
```

### Пояснение синтаксиса массовой вставки:

| Часть запроса | Описание |
|---------------|----------|
| `VALUES` | Ключевое слово перед списком кортежей значений |
| `(...), (...), (...)` | Каждый кортеж `(...)` — одна строка для вставки |
| `,` (запятая) | Разделитель между кортежами — **обязателен** после каждого, кроме последнего |
| `(...)` внутри кортежа | Значения для одной строки в порядке колонок |

### Ключевые моменты:

1. **Каждый кортеж в скобках** — это одна строка таблицы.
2. **Запятая между кортежами** — разделяет строки для вставки.
3. **Последний кортеж без запятой** — после последнего значения запятая не ставится.
4. **Единый запрос** — все строки вставляются одной операцией (быстрее, чем много отдельных INSERT).

### Пример с ошибками:

```sql
-- ❌ ОШИБКА: пропущена запятая между кортежами
VALUES 
    ('Товар 1', 'Описание', 'Категория', 100.00, 'SKU-001')
    ('Товар 2', 'Описание', 'Категория', 200.00, 'SKU-002');

-- ❌ ОШИБКА: лишняя запятая после последнего кортежа
VALUES 
    ('Товар 1', 'Описание', 'Категория', 100.00, 'SKU-001'),
    ('Товар 2', 'Описание', 'Категория', 200.00, 'SKU-002'),;

-- ✅ ПРАВИЛЬНО:
VALUES 
    ('Товар 1', 'Описание', 'Категория', 100.00, 'SKU-001'),
    ('Товар 2', 'Описание', 'Категория', 200.00, 'SKU-002');
```

## executemany в Python

В Python с модулем `sqlite3` для массовой вставки используется метод `executemany()`.

### Синтаксис:

```python
cursor.executemany("INSERT INTO Products (Name, Description, Category, Price, SKU) VALUES (?, ?, ?, ?, ?)", data_list)
```

### Пояснение:

| Часть кода | Описание |
|------------|----------|
| `cursor.executemany()` | Метод для выполнения одного запроса много раз с разными параметрами |
| `?` | Плейсхолдеры для параметров (защита от SQL-инъекций) |
| `data_list` | Список кортежей, где каждый кортеж — одна строка для вставки |

### Пример:

```python
import sqlite3

# Подключение к базе данных
conn = sqlite3.connect(r'C:\projects\Modul-3\VPc04_learning\learning.db')
cursor = conn.cursor()

# Список данных для вставки
products_data = [
    ('Беспроводные наушники', 'Высокое качество', 'Электроника', 2999.99, 'ART-1234-A'),
    ('Смартфон XYZ', 'Мощный смартфон', 'Электроника', 45999.00, 'ART-5678-B'),
    ('Книга по SQL', 'Руководство для начинающих', 'Книги', 899.50, 'ART-9012-C'),
]

# Массовая вставка
insert_query = """
    INSERT INTO Products (Name, Description, Category, Price, SKU)
    VALUES (?, ?, ?, ?, ?)
"""

cursor.executemany(insert_query, products_data)
conn.commit()  # Фиксация транзакции
conn.close()
```

### Преимущества `executemany()`:

1. **Безопасность:** Использование `?` защищает от SQL-инъекций.
2. **Производительность:** Все данные передаются одним вызовом.
3. **Читаемость:** Код чище, чем конструирование большого SQL-запроса.
4. **Удобство:** Не нужно вручную добавлять запятые и экранировать строки.

## Генерация случайных данных в Python

Для создания тестовых данных используем модуль `random`.

### Пример генератора:

```python
import random

# Списки для генерации
product_names = [
    "Беспроводные наушники", "Смартфон XYZ", "Ноутбук Pro", "Планшет Mini",
    "Умные часы", "Камера 4K", "Игровая консоль", "Монитор 27\""
]

categories = ["Электроника", "Книги", "Мебель", "Инструменты", "Спорт"]

descriptions = [
    "Высокое качество, гарантия 2 года", "Хит продаж 2024", 
    "Новинка сезона", "Премиум качество"
]

# Генерация одного товара
def generate_product(index: int) -> tuple:
    sku = f"ART-{random.randint(1000, 9999)}-{random.choice(['A', 'B', 'C', 'D'])}"
    name = random.choice(product_names)
    category = random.choice(categories)
    description = f"{random.choice(descriptions)}. Товар #{index + 1}"
    price = round(random.uniform(99.99, 99999.99), 2)
    return (name, description, category, price, sku)

# Генерация 1000 товаров
products = [generate_product(i) for i in range(1000)]
```

### Пояснение функций random:

| Функция | Описание | Пример результата |
|---------|----------|-------------------|
| `random.randint(a, b)` | Случайное целое число от a до b (включительно) | `random.randint(1000, 9999)` → `5432` |
| `random.uniform(a, b)` | Случайное число с плавающей точкой от a до b | `random.uniform(99.99, 99999.99)` → `45678.23` |
| `random.choice(list)` | Случайный элемент из списка | `random.choice(['A', 'B', 'C'])` → `'B'` |
| `round(x, 2)` | Округление числа x до 2 знаков после запятой | `round(45.6789, 2)` → `45.68` |

## Проверка вставленных данных

### SQL-запросы для проверки:

```sql
-- 1. Подсчитать количество записей
SELECT COUNT(*) AS total_products FROM Products;

-- 2. Показать первые 10 товаров
SELECT * FROM Products LIMIT 10;

-- 3. Показать только нужные колонки
SELECT Name, SKU, Price FROM Products LIMIT 5;

-- 4. Статистика по категориям
SELECT 
    Category, 
    COUNT(*) AS count, 
    ROUND(AVG(Price), 2) AS avg_price,
    MIN(Price) AS min_price,
    MAX(Price) AS max_price
FROM Products
GROUP BY Category
ORDER BY count DESC;

-- 5. Товары в ценовом диапазоне
SELECT Name, Price 
FROM Products 
WHERE Price BETWEEN 1000 AND 10000
ORDER BY Price DESC;

-- 6. Поиск по названию
SELECT * FROM Products WHERE Name LIKE '%наушники%';

-- 7. Найти товар по SKU
SELECT * FROM Products WHERE SKU = 'ART-1234-A';
```

### Пояснение операторов выборки:

| Оператор | Описание |
|----------|----------|
| `SELECT` | Выборка данных из таблицы |
| `COUNT(*)` | Подсчёт количества строк |
| `ROUND(x, 2)` | Округление числа x до 2 знаков после запятой |
| `AVG()` | Вычисление среднего значения |
| `MIN()` / `MAX()` | Минимальное / максимальное значение |
| `SUM()` | Сумма значений |
| `GROUP BY` | Группировка строк по значению колонки |
| `ORDER BY` | Сортировка результатов |
| `DESC` | Сортировка по убыванию |
| `ASC` | Сортировка по возрастанию (по умолчанию) |
| `LIMIT` | Ограничение количества строк в результате |
| `WHERE` | Фильтрация строк по условию |
| `BETWEEN` | Проверка попадания в диапазон |
| `LIKE` | Поиск по шаблону (`%` — любое количество символов) |
| `AS алиас` | Присвоение псевдонима колонке в результате |

## Удаление данных

```sql
-- Удалить все записи из таблицы
DELETE FROM Products;

-- Удалить записи с определённой категорией
DELETE FROM Products WHERE Category = 'Книги';

-- Сбросить автоинкремент (очистить последовательность ID)
DELETE FROM Products;
DELETE FROM sqlite_sequence WHERE name = 'Products';

-- Удалить таблицу полностью
DROP TABLE IF EXISTS Products;
```

### Пояснение операторов удаления:

| Оператор | Описание |
|----------|----------|
| `DELETE FROM` | Удаление строк из таблицы |
| `DELETE FROM ... WHERE` | Удаление только строк, удовлетворяющих условию |
| `sqlite_sequence` | Системная таблица SQLite для AUTOINCREMENT |
| `DROP TABLE` | Удаление таблицы вместе со структурой |

## Рекомендации

1. **Используйте `executemany()`** — самый быстрый и безопасный способ в Python.
2. **Обёрните в транзакцию** — `conn.commit()` после вставки для фиксации.
3. **Проверяйте уникальность SKU** — колонка `UNIQUE` вызовет ошибку при дубликатах.
4. **Используйте плейсхолдеры `?`** — защита от SQL-инъекций.
5. **Для больших объёмов** — разбивайте на пакеты по 1000-5000 записей.
